

## Проблема
Файлы для скачивания на странице «Аккредитация» имеют имена на кириллице:
- `Заявка_на_обяз_сертиф_продукции.docx`
- `Форма_подачи_Жалобы_обращения.docx`
- `Форма_подачи_Апелляции.docx`

Кириллица в URL без percent-encoding часто приводит к ответу 404 на статическом хостинге — отсюда сообщение «Файл недоступен на сайте».

## Решение
Переименовать файлы в латиницу и обновить три ссылки в `src/pages/Accreditation.tsx`.

## Изменения

### 1. Переименование файлов в `public/documents/`
| Было | Станет |
|---|---|
| `Заявка_на_обяз_сертиф_продукции.docx` | `zayavka-na-sertifikaciyu.docx` |
| `Форма_подачи_Жалобы_обращения.docx` | `forma-zhaloby.docx` |
| `Форма_подачи_Апелляции.docx` | `forma-apellyacii.docx` |

Атрибут `download` у `<a>` уже задан — при скачивании пользователь увидит именно это имя файла. При желании можно дополнительно прописать в `download="Заявка на сертификацию.docx"`, чтобы пользователь получил кириллическое имя при сохранении.

### 2. Обновление трёх ссылок в `src/pages/Accreditation.tsx`
- стр. 915 → `href="/documents/zayavka-na-sertifikaciyu.docx"`
- стр. 1201 → `href="/documents/forma-zhaloby.docx"`
- стр. 1382 → `href="/documents/forma-apellyacii.docx"`

Опционально добавить `download="<человекочитаемое имя>.docx"` чтобы файл сохранялся у пользователя с понятным русским именем.

## Что нужно после изменений
Нажать **Publish → Update**, чтобы обновлённый фронтенд (включая новые файлы в `public/`) попал на боевой домен. Без публикации файлы останутся доступны только в превью.

## Технические детали
- Lovable обслуживает `public/` как статические файлы.
- Имена файлов в `href` не проходят через `encodeURIComponent`, что и вызывает проблему доставки на проде с кириллицей.
- Атрибут `download` на `<a>` управляет только именем при сохранении, не самим путём запроса.
